# サブエージェントへ委譲する

## Overview

作業をサブエージェント・並列エージェント・下位モデルへ委譲するかどうかを判断し、委譲する場合に安全に実行するための手順です。
判断基準の根拠は `AGENTS.md` の「サブエージェント / マルチエージェントの使い方」です。
サブエージェント機能がないツールで作業している場合、この SOP は対象外です。

## Parameters

- **task** (required): 委譲を検討している作業内容。
- **reason** (required): 委譲理由。`コンテキスト汚染の分離` / `並列化` / `専門化` のいずれか。
- **success_criteria** (required): 完了と判定できる具体的な基準。
- **model_tier** (optional): 割り当てるモデル。機械的な収集・変換は下位モデル、判断を伴う実装・検証は上位モデル。

## Prerequisites

- `AGENTS.md` の「サブエージェント / マルチエージェントの使い方」を確認している。
- 委譲しなくても単一エージェントで完了できるかを先に検討している。

## Steps

### 1. 委譲するかを判定する

**Constraints:**

- MUST 委譲理由が「コンテキスト汚染の分離（目安 2,000 トークン以上の不要情報）」「並列化（互いに独立した作業）」「専門化（ツール・指示の分離が必要）」のいずれかに該当することを確認する。どれにも該当しない場合は委譲しない。
- MUST 順序依存・密結合・頻繁な状態同期が必要な作業は委譲しない。
- SHOULD トークン消費が 3〜10 倍になることに見合う効果があるかを確認する。

### 2. 分割の境界を決める

**Constraints:**

- MUST 「問題の種類」ではなく「コンテキストの境界」で分ける。
- MUST 同じ機能の実装・テスト・修正を段階別に別エージェントへ分けない。
- SHOULD 独立した調査パス、契約が明確な別コンポーネント、ブラックボックス検証の単位で分ける。

### 3. 指示を組み立てる

各エージェントに渡す指示に以下を含める。

- 目的と背景（そのエージェントが知るべき最小限）。
- 前提・制約（触ってよい範囲、守るルール、`AGENTS.md` など参照すべきファイル）。
- 成功基準（何をもって完了か）。
- 出力形式（要点サマリー。生ログや全文を返させない）。

**Constraints:**

- MUST 指示は自己完結にする。メイン会話の履歴を前提にしない。
- SHOULD ハンドオフの回数を最小にする。エージェント間の多段リレーを作らない。

### 4. 実行して結果を統合する

- 並列実行できるものは同時に起動する。
- 結果はサマリーで受け取り、矛盾があれば必要箇所だけ自分で確認する。

### 5. 検証する（成果物がある場合）

成功基準を明記した検証専用エージェントに確認させる。

**Constraints:**

- MUST 「一部のテストが通っただけで合格にしない。フルテストスイートを実行する」ことを指示に含める。
- SHOULD エッジケースと、失敗すべき入力（負のテスト）の確認を指示に含める。

## Verification

- 委譲理由が 3 条件のいずれかに該当していた。
- 各エージェントの成果が成功基準を満たしている。
- メイン会話に不要な生ログ・大量出力が持ち込まれていない。

## Rollback / Recovery

- サブエージェントの成果が基準を満たさない場合は、不足点を明記して同じエージェントに再依頼するか、単一エージェント作業に切り替える。
- 委譲によってかえって時間・トークンがかかっている場合は、途中でも単一エージェントに戻してよい。
