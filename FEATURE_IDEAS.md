# iKnoWay Personal Site Feature Ideas

作成日: 2026-05-30
対象: iKnoWay personal site

## 方針

このサイトは「同一コンテンツを複数テーマで見せる」ことと、アクセスするたびに変わるテーマガチャ体験が強い。
追加機能は、単に情報を増やすよりも「遊びに来る」「引いてみる」「眺める」方向に寄せると相性がよい。

実装面では、既存の `docs/shared/data.js` にデータを追加し、必要に応じて `docs/shared/*.js` で共通セクションとして注入する形が扱いやすい。

## おすすめ度が高い機能

### 1. 視聴ログ / 最近見たもの

アニメ、映画、ゲームの「最近見た・やった」を短文で並べる。
個人サイトらしさが出やすく、更新もしやすい。

案:

- セクション名: `Recent Activity`, `最近の摂取ログ`, `Now Watching`
- データ例: タイトル、種別、日付、ひとこと、タグ
- 表示位置: Profile の後、または Games の前

### 2. 推しキャラ / 推しポイント

作品一覧とは別に、好きなキャラや推しポイントを並べる。
アニメ・映画好きサイトとして自然で、コメントの個性も出しやすい。

案:

- セクション名: `Character Shrine`, `推しキャラ図鑑`, `推しポイント`
- データ例: キャラ名、作品名、好きな理由、属性タグ
- 表示形式: カード、図鑑、プロフィール風パネル

### 3. テーマ図鑑

実装済みテーマを一覧化し、「このサイト自体がガチャである」ことを見せる。
現在のサイト固有の面白さを一番伝えやすい。

案:

- 現在引いたテーマを表示
- 全テーマ一覧を表示
- `?theme=cyber` のような直接リンクを置く
- テーマごとに短い説明やレアリティ風ラベルを付ける

### 4. ランダム布教ボタン

ボタンを押すと anime / movies / games からランダムで1件おすすめを表示する。
静的サイトのまま実装でき、既存データを再利用しやすい。

案:

- ボタン名: `今日はこれ観ろ`, `Random Recommend`, `布教ガチャ`
- 表示内容: タイトル、コメント、タグ
- 表示位置: Hero、Anime セクション上部、または独立セクション

### 5. フィギュア欲しいもの / 眺めリスト

プロフィール内の「フィギュアのウィンドウショッピングが趣味」と相性がよい。
既存の Wishlist Web への導線にもできる。

案:

- セクション名: `Figure Watchlist`, `ショーケース候補`, `欲しいフィギュア`
- データ例: 名前、作品、状態、メモ、リンク
- 補足: 価格追跡ではなく、ゆるいメモとして扱うと静的サイト向き

## 追加カテゴリ案

### Now Watching

今見ているアニメ、積んでいる作品、次に見る候補を置くカテゴリ。

### Hall of Fame

人生ベスト級の作品だけを置くカテゴリ。
既存の anime / movies よりもさらに厳選した見せ方にできる。

### Quotes

好きなセリフや印象に残った言葉を短く置くカテゴリ。
引用は短めにし、コメント中心にすると扱いやすい。

### Theater Log

映画館で観た記録、IMAX メモ、劇場体験のひとことを残すカテゴリ。
「映画館派」というプロフィール要素と相性がよい。

### Music / OPED

好きなアニソン、劇伴、OP/ED を置くカテゴリ。
作品とは別の入口として機能する。

### Build Log

作った Web アプリや、このサイト自体の更新履歴を載せるカテゴリ。
Projects セクションとのつながりも作りやすい。

### Otaku Stats

視聴本数、泣いた回数、映画館派度、体力ゼロ度などをネタ統計として出すカテゴリ。
テーマごとに演出を変えると面白い。

### My Tags

`SF`, `電脳`, `美少女`, `バトル`, `泣く` など、タグから作品を逆引きするカテゴリ。
既存データの `tags` を活用できる。

## 優先度

最初に入れるなら、以下の順がよい。

1. テーマ図鑑
2. ランダム布教ボタン
3. 最近見たもの

理由:

- このサイト固有の「テーマガチャ」体験を強化できる
- 既存の `shared/data.js` を活かしやすい
- 静的サイトのまま実装できる
- 更新の手間が重すぎない

## 実装メモ

小さく始めるなら、まず `docs/shared/data.js` に以下のようなデータを足す。

```js
recommendPicks: {
  label: "今日はこれ観ろ",
  pools: ["anime", "movies", "games"]
}
```

または、最近のログ用に以下を追加する。

```js
recentActivity: [
  {
    type: "Anime",
    title: "作品名",
    date: "2026-05",
    comment: "ひとことメモ",
    tags: ["SF", "視聴中"]
  }
]
```

共通注入スクリプトを作る場合は、既存の `docs/shared/games.js` や `docs/shared/projects.js` と同じ方向で、
テーマ側の HTML を大きく変えずにセクションを追加すると管理しやすい。
