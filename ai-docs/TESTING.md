# TESTING.md

このファイルは、テスト方針、確認コマンド、手動確認ポイントを記録するためのドキュメントです。

## 自動更新ルール

以下が変わった場合は、このファイルを更新してください。

- 確認コマンドや起動手順を変えた。
- テスト方針、対象範囲、CI 設定を変えた。
- 重要な手動確認手順を追加・変更した。
- 既知の未テスト領域が増えた、または解消された。

## 確認コマンド

このプロジェクトに npm scripts / lint / 型チェック / 自動テストはない（素の HTML / CSS / JS）。
確認はローカルサーバーでの手動確認が基本。

| 目的 | コマンド | 備考 |
|---|---|---|
| ローカルサーバー起動 | `cd docs && python3 -m http.server 8080` | または `npx serve docs` |
| AI docs 鮮度チェック | `bash scripts/ai-docs-status.sh` | `--check` で CI 用 exit code |

## テスト方針

- 変更したテーマ・共通資産に関係する画面を優先して確認する。
- `docs/shared/` を変更した場合は、通常テーマ 2〜3 種 + 特殊構造テーマ（`fantasy-book` など）で確認する。
- `data.js` のスキーマを変えた場合は、全テーマの描画を一巡確認する。
- UI 変更は PC とモバイル幅、空状態、`prefers-reduced-motion` 有効時を確認する。

## 手動確認

| 機能 | 確認内容 | 手順 |
|---|---|---|
| テーマ表示 | 対象テーマが崩れず表示される | `http://localhost:8080/?theme=<id>` で固定表示 |
| テーマ切替 | 必ず別テーマに切り替わる | ページ内 Switch ボタンまたは `?switch=1` |
| ランダム選択 | ローダー経由で遷移する | 新しいタブで `http://localhost:8080/` を開く |
| コンテンツ描画 | `data.js` の内容が全セクションに反映される | ゲーム / アニメ / 映画 / 成果物 / 連絡先を目視 |
| コンソール | エラー・`console.log` 残りがない | DevTools コンソールを確認 |
| モバイル表示 | 横スクロールが出ない、文字はみ出しがない | DevTools のデバイスモード |
