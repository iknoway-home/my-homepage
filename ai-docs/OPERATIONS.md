# OPERATIONS.md

このファイルは、運用、リリース、デプロイ、復旧の入口です。

## 自動更新ルール

以下が変わった場合は、このファイルを更新してください。

- デプロイ方法、リリース手順、ロールバック手順を変えた。
- 本番環境、監視、バックアップの扱いを変えた。
- 障害対応や復旧手順を追加・変更した。
- 運用に使う外部サービスや権限を変えた。

具体的な手順が固定化したら `sops/*.sop.md` に切り出してください。

## 環境

| 環境 | 用途 | デプロイ方法 | 注意 |
|---|---|---|---|
| local | 開発 | なし（ローカルサーバーで確認） | `ai-docs/ENVIRONMENT.md` 参照 |
| staging | 検証 | なし | 検証環境は持たない |
| production | 本番 | `main` へプッシュ（GitHub Pages が `/docs` を自動公開） | プッシュ = 即公開。反映まで数分かかる場合あり |

## デプロイ手順

ビルドステップはない。詳細は `sops/release-or-deploy.sop.md` を参照。

```bash
# ローカル確認のうえで
git add <対象ファイル>
git commit -m "変更内容"
git push origin main
```

## ロールバック

| 方法 | 手順 | 注意 |
|---|---|---|
| コミットを打ち消す | `git revert <sha>` してプッシュ | 履歴を保ったまま戻せる。基本はこれ |
| 直前の状態に戻す | 問題のファイルだけ前バージョンに戻してコミット | 影響範囲を最小にできる |

## 監視・ログ

| 種類 | 場所 | 見るタイミング |
|---|---|---|
| デプロイ状況 | GitHub リポジトリの Pages / Actions タブ | プッシュ後に反映されないとき |
| 表示エラー | ブラウザの DevTools コンソール | 変更確認時 |
| アクセスログ | なし（解析ツール未導入） | — |

## バックアップ

| 対象 | 方法 | 頻度 | 復元手順 |
|---|---|---|---|
| サイト全体 | git 履歴（GitHub リモート） | プッシュごと | `git revert` / `git checkout <sha> -- <path>` |

## 障害対応

| 症状 | 初動 | 参照 SOP |
|---|---|---|
| 本番でテーマが表示されない | DevTools コンソール確認 → 直近コミットを疑い `git revert` | `sops/debug-or-fix-bug.sop.md` |
| Pages が反映されない | GitHub の Pages ビルド状況を確認 | — |
