# SECURITY.md

このファイルは、秘密情報、権限、入力検証、セキュリティ上の注意点を記録するためのドキュメントです。

## 自動更新ルール

以下が変わった場合は、このファイルを更新してください。

- 認証、認可、権限を追加・変更した（現状はなし）。
- 秘密情報、API キー、個人情報の扱いを変えた。
- 外部入力、外部 API 連携を追加した。
- セキュリティ上の制約、禁止事項、既知リスクが変わった。

秘密値そのものは書かないでください。

## 基本ルール

- 秘密情報をコミットしない（静的サイトのため、コミットしたものはすべて公開される）。
- 外部入力は境界で検証する（現状、ユーザー入力を受け取る機能はない）。
- 依存ライブラリを追加するときは、メンテナンス状況と用途を確認する（現状は外部依存ゼロ）。

## 秘密情報

| 種類 | 保存場所 | 参照方法 | 注意 |
|---|---|---|---|
| なし | — | — | API キー等は使用していない。追加する場合は静的サイトでは秘匿できない点に注意 |

## 認証・認可

なし（公開静的サイト。ログイン機能なし）。

## 入力境界

| 入力 | 検証場所 | 注意 |
|---|---|---|
| URL パラメータ（`?theme=` / `?switch=1`） | `docs/theme-router.js` | `THEMES` 配列に存在する id のみ受け付ける（ホワイトリスト方式を維持する） |

## 外部公開面

| 公開面 | 用途 | リスク | 対策 |
|---|---|---|---|
| GitHub Pages（`docs/` 全体） | サイト公開 | リポジトリの `docs/` に置いたものはすべて公開される | 内部ドキュメント・秘密情報を `docs/` に置かない（AI 用文書は `ai-docs/`） |
| メールアドレス（`data.js` の contact） | 連絡先表示 | スクレイピングによるスパム | 本人が公開を了承している情報のみ載せる |

## 既知リスク

| リスク | 影響 | 対応 |
|---|---|---|
| `docs/` への誤配置による内部情報公開 | 低〜中 | 配置ルールを `AGENTS.md` に明記。コミット前に `git status` で確認 |
